Overview

The CoronaKit Framework enables you to embed the graphical and interactive capabilities of Corona Corona into existing native apps. Specifically, you can wrap your Corona project into a native wrapper so that it can be added to the view hierarchy, much like adding a textfield or webview.

In this site, we will refer to this wrapper as a "Corona-based view" or "Corona view", because you can have multiple simultaneous Corona views at once.

SDK/Enterprise Compatibility

CoronaKit was designed to be added to existing native apps. Therefore — unlike CoronaSDK or CoronaEnterprise — CoronaKit cannot define or control the main application stack.

Because of this architecture, the following services are not available in CoronaKit:

  • [ads.*][api.library.ads]
  • [credits.*][api.library.credits]
  • [facebook.*][api.library.facebook]
  • [gameNetwork.*][api.library.gameNetwork]
  • [licensing.*][api.library.licensing]
  • [store.*][api.library.store]
  • Local and Remote Push Notification

The following have been removed to reduce binary size:

  • [lfs.*][api.library.lfs]
  • [socket.*][api.library.socket]

The following is currently not available and will be a subset of functionality due to the architectural limitations described above:

  • [native.*][api.library.native]

The following are temporarily not available:

  • [media.*][api.library.media]
